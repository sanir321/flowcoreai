WhatsApp Compliance Flow

Current vs. Proposed Architecture for GoWA Integration

Current Flow (Fragmented)

GoWA Webhook
1. Calculates 24h Window (in fire-and-forget task)
2. Calls Orchestrator
3. Waits for Response
4. Manages Presence (composing)
5. Waits for Artificial Delay
6. Sends Message via GoWA API
7. Manages Presence (paused)
Agent Orchestrator (Logic only, no sending)

Proposed Flow (PRD Aligned)

GoWA Webhook
1. Normalizes JID
2. Stores Inbound Message
3. Triggers Orchestrator (WaitUntil)
Agent Orchestrator (Full Ownership)
1. Strict 24h Window Check (Aborts if expired)
2. Analyzes Intent & Generates Response
3. Presence Update: composing
4. Artificial Typing Delay (35ms/char)
5. Dispatches to GoWA API
6. Presence Update: paused
7. Stores Outbound Message
A

Centralized Ownership

Move all compliance and sending logic to Agent Orchestrator as per PRD.

B

Shared Utility

Keep sending in Webhook/Actions but use a shared 'Compliance Dispatcher' utility.

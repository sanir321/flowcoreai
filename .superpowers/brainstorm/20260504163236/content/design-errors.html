Compliance Layer Design

Section 3: Error Handling & Testing

Error Handling

  • Window Expired: If >24h, we will create an entry in escalation_logs with trigger_type: 'wa_window_expired'. This alerts the user that they need to reply manually.
  • GoWA Timeout: If the GoWA server is down during presence/message sending, we log a warning but don't crash the orchestrator, ensuring the DB record of the AI response is still saved.

Testing Strategy

  1. Unit Test: Verify the 24h window calculation logic with mock timestamps.
  2. Integration Test: Mock the GoWA API and verify that /send/presence is called before /send/message.
  3. Manual Test: Send a message from a real phone and watch the "typing..." indicator.

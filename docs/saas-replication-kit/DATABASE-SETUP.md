# Database Setup: Schema & Triggers

To replicate FlowCore, you must apply the following core table structures to your Supabase PostgreSQL database.

## 1. Core Tables

### `workspaces`
Stores business-level configuration and branding.
- `id`: UUID (Primary Key)
- `name`: Text
- `welcome_template`: Text (The message sent to new customers)

### `conversation_sessions`
Tracks individual chat threads.
- `id`: UUID (Primary Key)
- `workspace_id`: FK to workspaces
- `customer_jid`: Text (The WhatsApp JID)
- `status`: enum (active, escalated, resolved, idle)
- `last_message_preview`: Text (For the sidebar)

### `messages`
Stores every inbound and outbound interaction.
- `id`: UUID (Primary Key)
- `session_id`: FK to conversation_sessions
- `content`: Text
- `direction`: enum (inbound, outbound)
- `role`: enum (customer, agent, assistant)
- `gowa_message_id`: Text (Unique for deduplication)

### `gowa_sessions`
Links your WhatsApp account to a workspace.
- `workspace_id`: FK to workspaces
- `gowa_session_id`: UUID (The ID generated by GoWA)
- `phone_jid`: Text (Your business WhatsApp number)

---

## 2. Important Indexes
Run these to ensure the dashboard stays fast:

```sql
-- For fast deduplication
CREATE UNIQUE INDEX idx_unique_gowa_msg_id ON messages (gowa_message_id) WHERE (gowa_message_id IS NOT NULL);

-- For multi-workspace routing
CREATE INDEX idx_gowa_sessions_lookup ON gowa_sessions (phone_jid, gowa_session_id);

-- For real-time inbox sorting
CREATE INDEX idx_sessions_last_message ON conversation_sessions (workspace_id, last_message_at DESC);
```

---

## 3. Real-time Enablement
You **MUST** run this command to allow the Inbox to update live:

```sql
ALTER PUBLICATION supabase_realtime ADD TABLE conversation_sessions, messages;
```
